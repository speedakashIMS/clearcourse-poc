## Overview

This component kit serves as the foundation for building consistent user interfaces across AEM implementations. It provides a set of well-tested, accessible, and maintainable React components that can be integrated into AEM projects to ensure design system consistency and development efficiency.

## Purpose

- **Consistency**: Ensure uniform UI components across all AEM projects
- **Reusability**: Provide a single source of truth for common UI patterns
- **Accessibility**: Build components with WCAG compliance in mind
- **Maintainability**: Centralize component updates and improvements
- **Developer Experience**: Simplify component integration in AEM projects

## Tech Stack

- **React**: Modern React with hooks and functional components
- **Tailwind CSS**: Utility-first CSS framework for rapid UI development
- **Vite**: Next-generation frontend build tool for fast development and optimized production builds

## Tailwind Configuration

This project uses design tokens from `design-tokens.css` to automatically generate Tailwind CSS theme variables. The build script processes these CSS variables and updates the `@theme` block in `src/index.css` with the design system values.

### Building Theme Variables

To update the theme variables in `index.css` from the design tokens:

```bash
npm run build:tailwind-config
```

This script processes CSS variables and adds them to the `@theme` block in `src/index.css` under the comment `/* Autogenerated from CSS variables - DO NOT EDIT MANUALLY */`.

### Workflow

1. Export design tokens from Figma as CSS variables to `design-tokens.css`
2. Run `npm run build:tailwind-config` to update theme variables
3. The script automatically processes and adds variables to `src/index.css`
4. Use the generated variables in your components

## Heroicons

This project includes [Heroicons](https://heroicons.com/), a set of free MIT-licensed SVG icons optimized for UI development.

### Usage

Heroicons provides three icon sets:

- **Outline icons** (`@heroicons/react/24/outline`) - 24x24 outline style
- **Solid icons** (`@heroicons/react/24/solid`) - 24x24 solid/filled style  
- **Mini icons** (`@heroicons/react/20/solid`) - 20x20 smaller solid style

#### Basic Example

```jsx
import { HeartIcon } from '@heroicons/react/24/outline';
import { StarIcon } from '@heroicons/react/24/solid';
import { CheckIcon } from '@heroicons/react/20/solid';

function MyComponent() {
  return (
    <div>
      {/* Outline icon */}
      <HeartIcon className="h-6 w-6 text-gray-500" />
      
      {/* Solid icon */}
      <StarIcon className="h-6 w-6 text-yellow-500" />
      
      {/* Mini icon */}
      <CheckIcon className="h-5 w-5 text-green-500" />
    </div>
  );
}
```

#### With Tailwind CSS

Icons work seamlessly with Tailwind CSS classes for sizing, coloring, and styling:

```jsx
import { ArrowRightIcon } from '@heroicons/react/24/outline';

function Button() {
  return (
    <button className="flex items-center gap-2 px-4 py-2 bg-blue-500 text-white rounded">
      Click me
      <ArrowRightIcon className="h-5 w-5" />
    </button>
  );
}
```

## Dos and Don'ts for AEM Transpilation

When building components that will be transpiled to AEM, it's important to understand what React features can be transpiled and what cannot. The transpilation process can handle React hooks and logic, but cannot transpile React DOM elements that don't exist in the AEM context.

### ✅ DO: Use React Hooks

React hooks are fully supported and will be transpiled correctly:

```jsx
import { useState, useEffect, useCallback, useMemo } from 'react';

function MyComponent() {
  const [count, setCount] = useState(0);
  const [isOpen, setIsOpen] = useState(false);

  useEffect(() => {
    // Side effects are transpiled correctly
    document.title = `Count: ${count}`;
  }, [count]);

  const memoizedValue = useMemo(() => {
    return expensiveCalculation(count);
  }, [count]);

  const handleClick = useCallback(() => {
    setCount(count + 1);
  }, [count]);

  return (
    <div>
      <button onClick={handleClick}>Count: {count}</button>
    </div>
  );
}
```

**Supported Hooks:**
- `useState` - State management
- `useEffect` - Side effects and lifecycle
- `useCallback` - Memoized callbacks
- `useMemo` - Memoized values
- `useRef` - Refs and DOM references
- `useContext` - Context API
- Custom hooks - Your own hook functions

### ✅ DO: Use Standard HTML Elements

Standard HTML elements are safe to use and will work in AEM:

```jsx
function MyComponent() {
  return (
    <div>
      <h1>Title</h1>
      <p>Paragraph text</p>
      <button onClick={handleClick}>Click me</button>
      <input type="text" />
      <a href="/page">Link</a>
      <img src="image.jpg" alt="Description" />
      <ul>
        <li>Item 1</li>
        <li>Item 2</li>
      </ul>
    </div>
  );
}
```

**Safe HTML Elements:**
- `div`, `span`, `section`, `article`, `header`, `footer`
- `h1` through `h6`
- `p`, `a`, `button`, `input`, `textarea`, `select`
- `img`, `svg`
- `ul`, `ol`, `li`
- `table`, `thead`, `tbody`, `tr`, `td`, `th`
- `form`, `label`, `fieldset`, `legend`

### ✅ DO: Use Functional Components

Functional components with props are fully supported:

```jsx
function Button({ text, variant, onClick }) {
  return (
    <button 
      className={`btn btn-${variant}`}
      onClick={onClick}
    >
      {text}
    </button>
  );
}

// Usage
<Button text="Click me" variant="light" onClick={handleClick} />
```

### ✅ DO: Use Conditional Rendering

All React conditional rendering patterns work:

```jsx
function MyComponent({ items }) {
  const [isVisible, setIsVisible] = useState(true);

  return (
    <div>
      {isVisible && <p>Visible content</p>}
      {items.length > 0 ? (
        <ul>
          {items.map(item => <li key={item.id}>{item.name}</li>)}
        </ul>
      ) : (
        <p>No items</p>
      )}
    </div>
  );
}
```

### ✅ DO: Use Event Handlers

Event handlers are transpiled correctly:

```jsx
function MyComponent() {
  const handleClick = (e) => {
    e.preventDefault();
    console.log('Clicked');
  };

  const handleChange = (e) => {
    console.log(e.target.value);
  };

  return (
    <div>
      <button onClick={handleClick}>Click</button>
      <input onChange={handleChange} />
    </div>
  );
}
```

### ❌ DON'T: Use React Router Components

React Router components like `Link` and `BrowserRouter` cannot be transpiled to AEM:

```jsx
// ❌ DON'T DO THIS
import { Link } from 'react-router-dom';

function Navigation() {
  return (
    <nav>
      <Link to="/page">Page</Link>  {/* This will NOT work in AEM */}
    </nav>
  );
}
```

**Instead, use standard HTML anchor tags:**

```jsx
// ✅ DO THIS INSTEAD
function Navigation() {
  return (
    <nav>
      <a href="/page">Page</a>  {/* Standard HTML works in AEM */}
    </nav>
  );
}
```

### ❌ DON'T: Use React Router Hooks

React Router hooks like `useNavigate`, `useLocation`, `useParams` are not available in AEM:

```jsx
// ❌ DON'T DO THIS
import { useNavigate, useLocation } from 'react-router-dom';

function MyComponent() {
  const navigate = useNavigate();
  const location = useLocation();
  
  // This will NOT work in AEM
}
```

**Instead, use standard browser APIs or AEM-specific navigation:**

```jsx
// ✅ DO THIS INSTEAD
function MyComponent() {
  const handleNavigation = (path) => {
    window.location.href = path;  // Standard browser navigation
  };

  return (
    <button onClick={() => handleNavigation('/page')}>
      Go to Page
    </button>
  );
}
```

### ❌ DON'T: Use React-Specific DOM Elements

Avoid React-specific DOM elements that don't exist in standard HTML:

```jsx
// ❌ DON'T DO THIS
import { Link } from 'react-router-dom';
import { NavLink } from 'react-router-dom';

function Navigation() {
  return (
    <Link to="/page">Page</Link>
    <NavLink to="/page">Page</NavLink>
  );
}
```

### ❌ DON'T: Use React Context for Routing

React Context providers for routing won't work in AEM:

```jsx
// ❌ DON'T DO THIS
import { BrowserRouter } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
      </Routes>
    </BrowserRouter>
  );
}
```

### ✅ DO: Use Standard HTML Forms

Standard HTML forms work perfectly:

```jsx
function ContactForm() {
  const [formData, setFormData] = useState({
    name: '',
    email: ''
  });

  const handleSubmit = (e) => {
    e.preventDefault();
    // Form submission logic
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={formData.name}
        onChange={(e) => setFormData({...formData, name: e.target.value})}
      />
      <input
        type="email"
        value={formData.email}
        onChange={(e) => setFormData({...formData, email: e.target.value})}
      />
      <button type="submit">Submit</button>
    </form>
  );
}
```

### ✅ DO: Use CSS Classes and Tailwind

CSS classes and Tailwind utilities work perfectly:

```jsx
function MyComponent() {
  return (
    <div className="container mx-auto p-4">
      <h1 className="text-headings-h1 font-bold text-grey-950">
        Title
      </h1>
      <button className="btn btn-light">
        Click me
      </button>
    </div>
  );
}
```

### ✅ DO: Use Props and Component Composition

Props and component composition are fully supported:

```jsx
function Card({ title, children, className }) {
  return (
    <div className={`card ${className}`}>
      <h2>{title}</h2>
      {children}
    </div>
  );
}

function App() {
  return (
    <Card title="My Card" className="card-large">
      <p>Card content</p>
    </Card>
  );
}
```

### Summary Checklist

**✅ Safe to Use:**
- React hooks (`useState`, `useEffect`, `useCallback`, `useMemo`, `useRef`, `useContext`)
- Functional components with props
- Standard HTML elements (`div`, `a`, `button`, `input`, etc.)
- Event handlers (`onClick`, `onChange`, `onSubmit`, etc.)
- Conditional rendering (`&&`, ternary operators)
- Array methods (`map`, `filter`, `reduce`)
- CSS classes and Tailwind utilities
- Component composition and props

**❌ Avoid:**
- React Router components (`Link`, `NavLink`, `BrowserRouter`, `Routes`, `Route`)
- React Router hooks (`useNavigate`, `useLocation`, `useParams`, `useSearchParams`)
- React Context for routing
- Any React-specific DOM elements that don't exist in standard HTML

### Best Practices

1. **Use standard HTML for navigation**: Always use `<a>` tags instead of React Router `Link` components
2. **Keep components self-contained**: Components should work independently without React Router context
3. **Use hooks for logic**: React hooks are safe and will transpile correctly
4. **Test in isolation**: Test components without React Router to ensure AEM compatibility
5. **Use browser APIs**: For navigation and routing, use standard browser APIs or AEM-specific solutions
